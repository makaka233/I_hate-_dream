Abstract—Multi-sensor and multi-user edge computing net-
works can support various data-intensive Internet of Things
(IoT) applications, which exhibit the characteristic of a task-data-
decoupled pattern. In this scenario, tasks generated by users can
be scheduled to edge servers (ESs), where the ESs compute the task
results and return them to the users. Meanwhile, a large number of
sensors collect and upload data to the ESs to meet the requirements
of task processing. However, existing works mainly consider the
task-data-coupled pattern, overlooking the cost associated with
sensor data collection processes. Therefore, we propose a joint
optimization problem involving service placement, task scheduling,
and resource allocation to minimize the total system cost, defined as
the weighted sum of delay and energy consumption of each user and
sensor. We jointly optimize service placement, user task schedul-
ing, transmit power allocation for sensors and users, computing
resource allocation for both the ESs and the cloud server (CS),
and transmission rate allocation for ES-ES and ES-CS connections.
Considering the differences in the update frequencies of the opti-
mization variables, we propose a dual-agent Deep Reinforcement
Learning (DRL) algorithm, which utilizes two SD3 (Softmax Deep
Double Deterministic Policy Gradients)-based DRL agents to make
service placement and task scheduling decisions asynchronously,
while embedding two optimization subroutines to solve the optimal
transmit power allocation, computing resource and transmission
rate allocations using numerical methods. The complexity and con-
vergence of the algorithm are analyzed and extensive experiments

I. INTRODUCTION
IN RECENT years, data-intensive Internet of Things (IoT)
applications have been widely deployed in areas such as
smart cities, intelligent transportation, and industrial automa-
tion [1], [2], [3]. These applications are based on a multi-sensor
and multi-user network architecture, where a large number of
sensor nodes and user devices are distributed across a wide
area. They also adopt a task-data-decoupled pattern, where
tasks are sourced from users and data is sourced from sensors.
Users generate computing tasks and offload them to servers for
processing, while sensors collect and upload data to the servers
to support user task processing.
Traditional cloud-computing-based approaches require all
user tasks and sensor data to be transmitted to the cloud server
(CS) for centralized processing, leading to several issues. First,
significant delays occur during both task and data transmis-
sion, resulting in degraded user experience. Second, massive
data transmission causes congestion in the backbone network,
thereby increasing the bandwidth burden. Third, data transmis-
sion and storage face serious security and privacy challenges.
Multi-access edge computing (MEC) [4], [5], [6], previously
known as mobile edge computing, has emerged as a promising
solution to these problems. MEC pushes the user task computing
andsensordatacollectionfunctionstoedgeservers(ESs)located
at the network edge.
To achieve high-efficiency multi-sensor and multi-user edge
computing networks, resource management plays a critical role
and addresses the following issues: 1) Service placement must
consider the requirements of both user task computing and
sensor data collection to place IoT services to the most suitable
ESs; 2) Task scheduling involves assigning user tasks to the most
appropriate ESs or the CS to minimize task processing delays;
3) Resource allocation encompasses transmit power allocation
for sensors and users, computing resource allocation for the
ESs and the CS, and transmission rate allocation for ES-to-ES

and ES-to-CS connections, with the goal of optimizing overall
system performance and ensuring efficient resource utilization.
However, existing works primarily adopt the task-data-
coupled pattern and overlook the complexities of multi-sensor
and multi-user IoT scenarios. These studies assume that the
tasks generated by users include both task requests and the data
required for task processing, neglecting the delay and energy
consumption costs associated with sensor data collection and
their subsequent impact on system performance. As a result,
these approaches are not suitable for data-intensive IoT ap-
plications, as they fail to make optimal resource management
decisions.
Furthermore, the update frequencies of different optimization
variables vary. Service placement involves steps such as service
container download and initialization, which incur significant
costs. Consequently, service placement decisions are not well-
suited for frequent updates. In contrast, task scheduling and
resource allocation decisions must be updated whenever users
generate new tasks. Most existing works assume that service
placement, task scheduling, and resource allocation decisions
are made simultaneously, which is not feasible in practical
systems.
To this end, in this paper, we propose a dual-agent Deep
Reinforcement Learning (DRL)-based service placement, task
scheduling, and resource allocation scheme for multi-sensor and
multi-user edge computing networks. The main contributions of
our work are summarized as follows:
1) We consider a task-data-decoupled approach in multi-
sensor and multi-user edge computing networks and for-
mulate a resource management problem that integrates
both user task computation and sensor data collection.
Specifically, we jointly optimizes service placement, user
task scheduling, transmit power allocation for sensors and
users, computing resource allocation for the ESs and the
CS, and transmission rate allocation for ES-ES and ES-CS
connections. Our goal is to minimize the total system cost,
which is the weighted sum of the delay and the energy
consumption of each user and sensor.
2) Considering the high complexity of the optimization prob-
lem and the different update frequencies of the optimiza-
tion variables, we design a dual-agent DRL algorithm
basedonSD3(SoftmaxDeepDoubleDeterministicPolicy
Gradients). In our algorithm, a service placement agent
makes service placement decisions, while a task schedul-
ing agent handles task scheduling decisions. They work
collaboratively in two different frequencies. Two subrou-
tines are embedded to solve the optimal transmit power
allocation for sensors and users, the optimal computing
resource allocation for the ESs, and the optimal transmis-
sion rate allocation for ES-ES and ES-CS connections.
This hybrid approach effectively reduces search space of
the SD3 algorithm and achieves faster convergence.
3) We analyze the complexity of our algorithm and evaluate
its convergence. Then, we conduct extensive experiments
by varying 8 crucial parameters. The experimental results
demonstrate the superiority of our scheme compared to
three other reference schemes.
The rest of this article is organized as follows: Related work is
discussed in Section II. The system model and problem formu-
lation are described in Section III. In Section IV, we provide a
detailed description of our proposed algorithm. We evaluate the
proposed algorithm in Section V and discuss the results. Finally,
we summarize our work in Section VI.
II. RELATED WORKS
MEC pushes computing functions to the edge of the net-
work, enabling users to offload computation-intensive tasks to
the ESs equipped on/near network edge nodes, such as base
stations (BSs), thereby enhancing task processing capabilities.
Service placement and task scheduling are two critical research
challenges in MEC. In recent years, a substantial amount of
research has focused on the service placement and task schedul-
ing problems, as well as their joint optimization [7], [8], [9],
[10], [11], [12]. The work in [8] addressed the online service
placement and request allocation problem in MEC networks.
In this study, service requests are allocated to cloudlets, and
effective prediction mechanisms are employed to pre-install ser-
vice instances and revoke idle instances, aiming to maximize the
profit for service providers. The work in [9] recognized that data-
intensive services require not only computing resources but also
storage resources for server code and data, as well as network
bandwidth for receiving data from users. The authors proposed a
framework to jointly optimize service (code and data) placement
and request scheduling, considering storage, communication,
computing resources, and budget constraints. In [10], the authors
highlighted that services like augmented reality require users to
upload data for service execution and then download the output
for user utilization. This bidirectional communication may be
asymmetric, occupying different portions of the BS’s uplink and
downlink bandwidth capacity. The study investigated the joint
optimization problem of service placement and request routing
under multidimensional constraints and proposed an algorithm
utilizing stochastic rounding techniques to achieve near-optimal
performance.
In MEC, both edge-edge collaboration and edge-cloud col-
laboration enable different ESs and the CS to share resources,
thereby expanding the range of services available and realizing
load balance. A considerable body of work has focused on the
collaboration [13], [14], [15]. In [13], the authors proposed a
scheme in which tasks offloaded to an RSU can be further
offloaded to other RSUs, thus the tasks and task results can
be flexible processed and returned to vehicles based on their
cross-RSU mobility and balance the loads among RSUs. In [14],
task scheduling in collaborative edge computing networks was
discussed, with the solution allowing tasks to be offloaded to
other edge nodes when the load on a given node becomes exces-
sive. The authors employed Lyapunov optimization to ensure
the long-term stability of the task queue. In [15], a resource
optimization framework was developed that takes into account
user preferences in edge node resource management, effectively
addressing the challenge of excessive resource allocation.
Many existing studies focuses on joint sensing and task of-
floading design in MEC networks [16], [17], [18]. The authors

The above studies mainly consider the transmission optimiza-

ignoring the interaction between user task processing and sensor
data transmission. On the other hand, many researches on joint
task scheduling and transmission in MEC network primarily
consider a task-data-coupled pattern, where computing tasks
generated by users are assumed to include both the task requests
and the data required for processing. In this model, once the
ESs and the CS receive the tasks offloaded from users, they
can perform task computation based on the data embedded
within the tasks. However, in multi-sensor, multi-user edge
computing networks, data-intensive IoT applications typically
adopt a task-data-decoupled pattern. In this pattern, the ESs and
the CS perform task computation only when both the user tasks
are offloaded and the sensor data are transmitted to them. This
introduces significant complexity to resource management, as
the sensor data uploading processes are coupled with task com-
puting, and they must be considered jointly in the optimization
of service placement, task scheduling, and resource allocation.
In particular, both the delay and energy consumption costs
associated with user task processing and sensor data collection
need to be minimized.
Moreover, the joint optimization problems discussed above
generally assume simultaneous decision-making. However, this
approach is challenging in practical systems, where service
placement decisions are typically updated on a much slower time
scale (e.g., every few hours or days) compared to task scheduling
and resource allocation decisions, which are updated much
more frequently (e.g., every few seconds). In [23], the authors
introduced a dual time-scale framework to address the joint
service placement and task scheduling problem, with the goal of
maximizing network utility while maintaining the stability of the
entire edge network. Similarly, [26] proposed a dual time-scale
approach for caching and scheduling, which minimizes the
weighted sum of energy consumption while subject to storage
and delay constraints. In [24], the authors designed a hierar-
chical DRL-based collaborative service caching and scheduling
strategy, aiming to minimize delay and energy costs. However,
these studies did not account for caching costs or communi-
cation resource allocation. In [25], the authors explored a dual
time-scale strategy for joint service placement, task scheduling,
and resource allocation to minimize system costs, though their
approach is limited to multi-user, single-edge scenarios. Overall,
none of these works consider the task-data-coupled pattern.

A. Network Architecture
As depicted in Fig. 1, we consider a multi-sensor, multi-user
edge computing network with both edge-edge and edge-cloud
collaborations. In this scenario, there are M BSs, each equipped
with an ES. Communication between BSs is enabled through
wired connections within the access network, facilitating edge-
edge collaboration. The CS is located behind the core network,
and communication between the ESs and the CS is achieved
via wired connections between the access and core networks,
enabling edge-cloud collaboration. Within the coverage area of
each BS, there are two types of terminal devices:
1) Sensors, which collect data required for services and up-
load it to the BSs where the corresponding services are deployed.
2) Users, which generate various types of tasks and offload
them to the BSs for processing. Communication between termi-
nal devices and BSs occurs via a cellular wireless network.

D. Dual-Agent DRL Algorithm Design
In practical systems, optimization variables are updated at
different frequencies depending on their roles. For example,
service placement involves processes such as downloading and
initializing service containers, which incur significant costs. As
a result, frequent updates to service placement decisions are
not practical. On the other hand, task scheduling and resource
allocation need to be updated continuously, as new tasks are
generated by users.
To address this, we first model the solving process of our
optimization problem as two update periods. As illustrated in
Fig. 3, the update frequency for service placement decisions is
relatively long, with the update period denoted by ti
, where i ∈
{1, 2, . . . }. Each long update period consists of T short periods,
during which task scheduling and resource allocation decisions
are made at the beginning of each short period, represented by
ti
j, where j ∈ {1, 2, . . . , T}.
To address the complex optimization challenges introduced
by the dual-timescale, we propose a dual-agent DRL algorithm.
Specifically, we employ a service placement agent based on
SD3 with a long update frequency to make service placement
decisions, a task scheduling agent based on SD3 with a short
update frequency to handle task scheduling, and two subrou-
tines, also with short update frequencies, to perform transmit
power allocation, computing resource allocation, and transmis-
sion rate allocation using numerical methods. By integrating this
dual-agent collaborative architecture with optimization subrou-
tines, the proposed algorithm can effectively address the dual-
timescale optimization problem. The two agents with different
update frequencies, interact to acquire more comprehensive
state information and achieve optimal decision-making through
cooperative training.
1) Architecture of the SD3 Algorithm: SD3 is an enhanced
DRL algorithm based on Twin Delayed Deep Deterministic
Policy Gradient (TD3) [37], which is itself an improvement over
Deep Deterministic Policy Gradient (DDPG) [38]. SD3 adopts
the softmax operator in updating value functions within actor-
critic methods, and it is tailored to effectively handle large action

E. Algorithm Deployment
The proposed algorithm includes a device transmit power al-
location algorithm and a dual-agent DRL algorithm. The device
transmit power allocation algorithm is implemented on each user
and sensor, while the dual-agent DRL algorithm operates on the
centralized network controller.
At the beginning of each time slot, each sensor and user
collects and stores the channel state information between them-
selves and their affiliated ESs. Then, the device transmit power
allocation algorithm determines the transmit power allocation
for each device based on the information of user tasks and sensor
data. Users and sensors then transmit their tasks and data to the
affiliated ES according to the allocated transmit power.
The dual-agent DRL algorithm is deployed on the centralized
network controller. At the start of each time slot, the network
controller collects and stores the current system state, including
the available computing and communication resources of each
ES. Then, the network controller optimizes service placement
and task scheduling by executing the algorithm. If the current
time aligns with a service placement period, the algorithm
first makes service placement decision, updating the deployed
service instances on each ES. It then determines which ES or CS,
hosting the relevant service, will process the user tasks. Subse-
quently, the algorithm allocates transmission rates between the
ESs and between ESs and CSs. The ESs transmit user tasks
and sensor data according to the allocated transmission rates.
Lastly, the algorithm allocates computing resources to each ES
and CS, ensuring the required resources are provided to process
user tasks.

C. Comparative Schemes and Performance
To effectively evaluate the performance of the proposed algo-
rithm, we compare it with three alternative schemes:
1) Greedy Service Placement (GSP): In this scheme, service
placement decisions are made greedily based on the number of
scheduled tasks and the amount of uploaded data. Specifically,
the service corresponding to the highest number of scheduled
tasks and uploaded sensor data is placed on the respective ES.
Task scheduling decisions are determined using DRL, following
a similar approach to [40]. The transmit power allocation for
users and sensors, transmission rate allocation for ESs, and
computing resource allocation are consistent with those in the
proposed scheme.
2) Equal Resource Allocation (ERA): This scheme allocates
the transmission rates and computing resources of ESs equally
based on the number of scheduled tasks and the amount of
uploaded data. The service placement decisions, task scheduling
decisions, and transmit power allocation decisions are identical
to those in the proposed scheme.
3) Fixed Power Allocation (FPA): In this scheme, users and
sensors transmit at their average power levels. The service
placement decisions, task scheduling decisions, transmission
rate allocation, and computing resource allocation are the same
as those in the proposed scheme.
We evaluate the performance of the proposed scheme and
the three comparative schemes by varying 6 key parameters.
For each configuration, we conduct 20 independent experiments

to the other three schemes, the proposed scheme consistently
achieves better performance across multiple scenarios. It ef-
fectively reduces system costs, particularly in multi-sensor and
multi-user edge computing networks.

VI. CONCLUSION
Inthispaper,weinvestigatenetworkresourcemanagementfor
data-intensive IoT applications in multi-sensor and multi-user
edge computing networks. Our objective is to minimize the
system cost, which is a trade-off between total delay and energy
consumption of all users and sensors. The key distinction of this
work from most existing studies is that we consider a task-data-
decoupled pattern and jointly optimize service placement, task
scheduling, and resource allocation for user task processing and
sensor data collection. We propose a dual-agent DRL method
based on the combination of the SD3 algorithm and multiple
numerical methods, facilitating asynchronous updates of the op-
timizing variables. We analyze the complexity and convergence
of our algorithm. Then, extensive experiments are conducted,
where we compare our scheme with three other schemes across
multiple scenarios. The results demonstrate the superiority of
our approach.
